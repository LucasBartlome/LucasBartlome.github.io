Project 1: AI Podcast Generator

Worked with a team at Motivation Technology to create an AI-based Podcast Generator prototype loosely based on Google’s NotebookLM project. The project used Python within a VS Code environment, and OpenAI’s ChatGPT and TTS (Text to Speech) API call to create the Podcast scripts. A simple UI was created to allow granular control of the parameters used to create podcasts (e.g. Title and Episode, names of hosts, voices used). The output was an MP3 file that could be played in a simple web-based player.

  • OpenAI GPT and Text-to-Speech APIs
  • Python and Visual Studio Code
  • Use AI to create podcast script using GPT APIs and Outputs in JSON format
  • Took advantage of OpenAIs TTS to create mp3 fragments

The GUI when the podcast generator is opened allowing you to select a project folder

This is the main menu GUI Where you are able to select all of the variables of your podcast such as The name of you podcast, the voices of the hosts, and the style of podcast(Serious, Funny, etc.) here is also where you decide what you want the topic of your podcast to be!

When choosing your podcast topic you have a couple of options, you can easily enter a prompt (shown below), or you can enter a file that OpenAIs GPT will scrape and attempt to find the most important information (shown above), Or you can even enter a website URL that will get scraped and use that information as the podcast topic.

After running the program it gives you an MP3 File of your fully completed AI generated Podcast!

Project 3: Database In MySQL

Collaborated on a team project to design and implement a database in MySQL for a test company called Phoenix Corp. The goal was to track all employees and products in a simulated store environment in order to identify meaningful patterns within the data. We started by designing an entity-relationship diagram (ERD) to map out the structure of the database, which included roughly 10 entities. Two of these sets were modeled using inheritance—one for different employee types and another for different product types. Using the ERD as a guide, we built the database schema and populated it with realistic testing data. To demonstrate the value of the system, we developed 15 SQL queries that incorporated joins, aggregate functions, subqueries, and more to extract insights and highlight specific business cases within the data. This experience strengthened my skills in relational database design and advanced query writing, while also giving me valuable perspective on how businesses leverage databases to analyze operations and drive data-driven decisions.
